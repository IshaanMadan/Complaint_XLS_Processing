Name the Sheet Primary Sheet is to be named "Source"
Date Fields to be Formatted Date fields within the extract are to be identified and formatting to be done to keep the dates in dd/mm/yyyy format.
Quality classification to be scrubbed "Quality Classification" column within the extract to be identified. Data in Quality Classification will have 5 levels, each level seperated by a "|". Separate columns are to be created for each Level and are to be named - "Division", "Franchise", "Product Family", "Product"," Quality Site" respectively.
Open Case Age Buckets to be Formed "Report Run Date" column to be created with the date on which the report is run. A column named "Case Age" to be created to find the difference between "Date Originated" field and "Report Run Date" fields. Then the values are to be grouped in another column named "Case Age Buckets" as following - Below 45 days, 45 to 90 Days, 90 to 120 Days and Above 120 Days. Note: This Acitivity is to be performed only on records still open. Compliant Status is not equal to Closed.
Time to Failure to be calculated "Time to Failure" column to be created and a comparison of "Installed on Date" Field to "Date Orignated Date" be performed to give "time to failure" date. If "Install Date" is blank, fill values from "Shipped On Date". "Time to Failure" column to be created and a comparison of "Installed on Date" Field to "Date Originated Date" be performed to give "time to failure" date. Then, "Time to Failure Buckets" column to be created with Less than "90 Days, 90 to 365 Days, 1 to 2 Years, Above 2 Years" classifications
Install Complaints to be identified The "SR Type" column is to be searched for the word "install"and if found those records are to be flagged as an installation Complaint. A new column named "Installation complaint" to be created with Yes/No values.
Complaint Regions to be Mapped If "Account Country" is blank fill values from "Event Country". Master list of regions to be mapped with the Field named "Account Country". Separate Column named "Regions" to be created to capture this information.
QA As Reported Code Formatting Once the above Tasks(1-7) are complete, a duplicate tab of the "Source" sheet is to be created and named as "As Reported Source" . Formatting for this task is to be performed on the duplicate sheet and the original "Source" sheet is to be protected from this task. "QA As Reported Code" column within the extract is to be identified. Each row can have multiple Codes. In case of Multiple codes, each code will be seperated by ";". Formatting is to be done to make each row contain only one code.Each code will have 5 levels, each level seperated by a "|". Separate columns are to be created for the last 2 levels of the code and be named as "As Reported Code" .
